# BOM & Sourcing

This directory manages the Bill of Materials (BOM) process for Tznami keyboard PCBs and complete builds — from KiCad BOM export through to final component ordering.

---

## BOM Generation

### KiCad Native BOM Export
KiCad can export a BOM directly from the schematic via *Tools → Generate BOM*. The default output is a CSV. Add custom fields in the symbol editor for `LCSC`, `MPN`, `Manufacturer`, and `Description` — these will appear in the exported BOM.

**Recommended KiCad BOM plugin**: `bom_csv_grouped_by_value_with_fp`  
This plugin groups identical components (same value + footprint) and outputs one row per unique part, greatly simplifying ordering.

### KiBot BOM (Preferred)
KiBot generates a much cleaner BOM with filtering, grouping, and LCSC part number columns. It also generates iBOM HTML alongside the CSV. Run via `kibot -c kibot.yml`. See `../pcb-design/kibot-config-template/`.

## Suppliers

### LCSC (Primary for SMD passives)
**URL**: https://lcsc.com  
Best source for passives, diodes, MCU, and USB connectors. Competitive pricing, integrates directly with JLCPCB SMT assembly (they share inventory). Use LCSC part numbers (e.g., `C14663`) in your KiCad symbol fields.

### Digi-Key (Backup / Specialty)
**URL**: https://digikey.com  
Excellent stock reliability, fast US shipping. Use when LCSC is out of stock on critical components. More expensive than LCSC but worth it for guaranteed supply in small runs.

### Mouser Electronics
**URL**: https://mouser.com  
Similar to Digi-Key. Strong on ST Microelectronics products (STM32 family). Good for exact-spec crystals.

### Octopart (Multi-supplier Search)
**URL**: https://octopart.com  
Cross-references a part number across all major distributors simultaneously. Essential for finding best price/stock when a part is scarce.

### KBDfans / Mkultra / Deskeys (Switch & Stab Sourcing)
For switches, stabilizers, and hotswap sockets:
- KBDfans: https://kbdfans.com
- 3DKeebs: https://3dkeebs.com
- Mkultra: https://mkultra.click
- Divinikey: https://divinikey.com (US, fast shipping)

## BOM Management Workflow

1. **Design phase**: Add LCSC/MPN fields to all KiCad symbols during schematic capture.
2. **KiBot run**: Auto-export BOM CSV on every CI build (see `shared/integrations/`).
3. **Review**: Open BOM CSV in Excel/Google Sheets, check quantities, verify part availability at LCSC.
4. **Octopart check**: Any parts not stocked at LCSC → search Octopart for alternatives.
5. **Order**: Place LCSC order; for JLCPCB SMT assembly, export the JLCPCB-format CSV (CPL + BOM).
6. **Receive**: Update `bom-template.csv` with actual unit costs and lot sizes received.

## iBOM (Interactive BOM)

iBOM generates an HTML file with the PCB layout + schematic data embedded. Click any component in the list and it highlights on the PCB silkscreen view. Invaluable for:
- Hand assembly (know exactly where each component goes)
- QA inspection (verify correct component placement)
- Explaining the board to new team members

Generated by KiBot; also available as a standalone KiCad plugin:  
https://github.com/openscopeproject/InteractiveHtmlBom
